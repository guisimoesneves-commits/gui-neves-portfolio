FNZ AMS HUB

Role Product designer
Timeline 10 Months

The project

AMS HUB empowers investment advisors to efficiently build, optimize, and analyze fixed-income portfolios through a unified platform.

The Asset Management Solutions platform (AMS HUB), originally known as HUB, was initially developed by the startup YieldX to support investment advisors. The platform enables users to create, optimize, and analyze fixed-income investment portfolios. Its core functionality is structured around a suite of integrated applications within the platform, including AssetExplorer, InPaaS, BestFit, Optimizer, and LadderBuilder.

With the product’s natural evolution and its acquisition by FNZ, it became necessary to transform HUB into a more comprehensive and sophisticated account management platform. Initially, HUB focused primarily on account optimization and management features; however, this scope did not fully align with broader market expectations. In response, the design and product teams defined a new vision—HUB 2.0—structured around four core entities: proposals, accounts, models, and households. This shift required not only redefining how data and relationships were managed, but also rethinking the platform’s navigation, architecture, and component system.

The challenge ultimately involved rebuilding the platform from the ground up to support this new conceptual model while ensuring consistency and scalability. A key objective was to develop a flexible white-label solution, enabling clients to customize the interface according to their own brand identity without compromising usability or cohesion.

The process

The first step in redesigning the platform was to conduct a detailed analysis of the existing version. We began by mapping the navigation structure to identify opportunities for improving the user experience and incorporating new features. Using Miro as our documentation tool, we reviewed all pages across each user journey, allowing us to identify patterns and ensure consistency throughout the platform.

With the navigation structure thoroughly analyzed and key inconsistencies and patterns identified, we moved into the wireframing phase to validate our hypotheses and define the future experience. We began by designing low-fidelity wireframes for the platform’s navigation, focusing on improving clarity, hierarchy, and scalability across different user journeys. Next, we developed wireframes for the application’s core workflows and wizards, ensuring that complex tasks could be completed in a guided, intuitive manner. Finally, we focused on structuring the platform’s newest feature—the entity-based model—translating the concepts of proposals, accounts, models, and households into clear, usable interfaces that aligned with both user needs and business requirements.

The solution

Following this initial analysis, we identified the key improvements required and documented them, prioritizing each initiative within a Jira board. At the same time, given the platform’s need to support a white-label approach—and the significant gap between the existing design system and its implementation—we recognized the need to establish a new, unified design system to ensure consistency from the ground up.

As a result, the solution was structured into two parallel tracks: the redesign of AMS HUB, encompassing navigation, interactions, and overall user experience; and the creation of Miles, a new design system designed to provide consistency, scalability, and seamless alignment between design and development.

Target Model Builder

Target Model Builder is a fixed income fund based app, responsible for creating proposals based on a specific target determined by the user. These objectives can be: Yield, Risk, or Income. It was the old InPaaS that was redesigned for the AMS HUB project.

SMA Builder

SMA Builder is a fixed income based app, responsible for creating proposals and models based on a set of constraints determined by the user. It was the old BestFit that was redesigned for the AMS HUB project.

LadderBuilder

LadderBuilder is a fixed income based app, responsible for creating proposals and models using a laddering strategy, which consists on buying and selling bonds when they reach their maturity date.

AssetExplorer

AssetExplorer is an app that allows the user to create fully customized proposals and models. It is dedicated to a more experienced user with total domain over the assets available on the market. With AssetExplorer, the user is able to select an asset from all three different types available on the platform: Fixed Income, Funds, and Equities.

Optimizer

Optimizer is an app that allows users to optimize existing accounts based on a determined set of constraints. The account performance can be improved with just a couple of clicks.

Proposals

With AMS HUB, proposals started to be the main source of management on the platform. The objective of this entity is to allow the advisor to have a way to validade the investment idea with their client before converting it on an account. Until that is done, the users can change the proposal as many times as they see fit.

The Design System - Miles

One of the platform’s key requirements was to support a white-label approach. To address this, we collaborated closely with the development team to evaluate whether to build a design system from scratch or adopt an existing library as a foundation. After several rounds of discussion, we selected Chakra UI as the base for Miles, as it provided the flexibility and scalability needed to support customization across different clients.

With this decision in place, we conducted a comprehensive component mapping exercise to identify which elements could be reused from the existing platform and which needed to be newly designed alongside AMS HUB. This mapping was then aligned with the Chakra UI library, allowing us to clearly distinguish between components to be customized and those to be built from scratch, ensuring efficiency and consistency throughout the system.

Miles was developed following atomic design principles and structured using design tokens, enabling a highly scalable and customizable system with low maintenance overhead. This approach ensured strong alignment between design and development while reducing long-term effort for both teams.

The name “Miles” was inspired by Miles Davis, one of the most influential musicians of the 20th century, reflecting the system’s emphasis on creativity, structure, and enduring impact.

Core

The core, as the name says, is the main part of the design system, every component and basic styles are present in this library session. All the other sessions are based on this document.

Context

Context consists of more complex components built upon the core foundation, and is responsible for supplying structure and data to subsequent layers of the system.

Modules

The Modules section comprises higher-level components that often function as near-complete page structures. Modules are only created when there is a clear need for reuse across multiple areas of the platform, ensuring consistency while minimizing maintenance effort.
